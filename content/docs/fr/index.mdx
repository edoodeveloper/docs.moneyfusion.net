Cette documentation vous guidera dans l'intégration des API de Fusion Pay dans votre site e-commerce et/ou votre application mobile.

## Prérequis

### Compte Money Fusion Enterprise

Vos informations, saisies lors de la création du compte, servent de référence numérique au sein des systèmes Money Fusion. Nous utilisons ces informations pour identifier votre compte et les applications que vous créez.

Ces informations sont nécessaires à des fins de vérification.

## Pour commencer

<Steps>
<Step>
### Configuration du compte

- Assurez-vous d'avoir un compte Money Fusion
- [Créez-en un](https://moneyfusion.net/login?tab=signup) si nécessaire

</Step>

<Step>

### Accès au compte

<Steps>
  <Step>
    Connectez-vous à [votre compte](https://moneyfusion.net/dashboard)
  </Step>
  <Step>
    Cliquez sur ["Fusion Pay"](https://moneyfusion.net/dashboard/fusionpay) dans
    le menu de gauche.
  </Step>
</Steps>

</Step>
<Step>
### Création d'une application

![Illustration création d'application](https://sc-digital.nyc3.cdn.digitaloceanspaces.com/sc-digital/images/1770812734849-Capture_d_____cran_2026-02-11_122044.png)

Un **lien API** sera généré après validation. Il permettra à votre serveur d’effectuer des requêtes vers l’API Fusion Pay.

<Callout type="warning" title="Règles de sécurité">

- L’URL du site doit être valide et publiquement accessible.
- Le `callback_url (facultatif)`. URL de redirection après paiement si disponible doit appartenir au domaine principal ou à un sous-domaine autorisé.
- Seules les adresses IP enregistrées pourront consommer l’API.
- Toute IP non déclarée sera automatiquement bloquée.

</Callout>

<Callout type="idea" title="Astuce pratique">

Si vous tentez un paiement depuis une IP non autorisée, vous recevrez le message suivant :

**IP non autorisée ou application non approuvée (xxx.198.xx.252)**

L’IP affichée entre parenthèses est **celle que vous devez enregistrer** dans votre application Fusion Pay.

</Callout>

</Step>

</Steps>
